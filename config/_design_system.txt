HRM System
Design System & Style Guide
Odoo Hackathon 2026  |  Based on EmPay + Odoo 17 UI Patterns
1. Brand Identity
The HRM system inherits its visual identity from Odoo's enterprise product suite, adapted for the EmPay payroll module. The brand is defined by a single primary color — plum purple — layered over a minimal neutral base.
1.1 Brand Principles
Clinical clarity — information-dense layouts, zero decorative elements.
Neutral canvas — white and near-white surfaces let data take center stage.
Single accent — purple is the only strong color; all else is gray or semantic.
Precision borders — 0.5px strokes, never drop shadows.

Available views: Kanban (grid), List (rows), Activity (timeline), Graph (chart), Pivot (table), Hierarchy.

7. HRM Module Patterns
7.1 Payslip Form Layout
Header block: 2-column grid — Employee Details (left) | Pay Period (right).
Employee, Contract, Pay Run fields on the left.
Period Start and Period End fields on the right.
Tab strip below the header: 'Attendance Summary' | 'Salary Breakdown'.
Attendance tab: 3-column stat grid (Present, Half, Leave Days) + (Absent, Total Working Days).
Action bar order: [Confirm] [Cancel] [Recompute] [Print Payslip] | [Draft ▶ Done].
7.2 Employee List / Kanban View
Kanban default. 2-column grid on desktop, 1-column on mobile.
Cards show: avatar, name, job title, email, phone, tags/chips, status icon.
Status icons (top-right corner): house (WFH), plane (travel/leave), location pin (on-site), clock (pending).
New button: purple filled, top-left above the grid.
Pagination: 'X-50 / 87' counter with prev/next arrows, top-right.
Department filter sidebar persistent on left in both list and kanban views.
7.3 Odoo Platform Pricing Context
For the Indian market, Odoo One pricing is 580 Rs/month for ALL apps. The platform positions itself as an affordable all-in-one solution. The HRM module is part of the Employees app which covers: hiring, onboarding, attendance, performance, payroll, and employee self-service — all AI-powered.
8. Do & Don't
DO
DON'T
Use 0.5px borders throughout
Use drop shadows or gradients
Use font weights 400 and 500 only
Use font-weight 600 or 700
Use semantic colors for stat values
Color numbers for decoration
Keep the purple accent to 1 purpose per view
Use purple for multiple unrelated elements
ShadingType.CLEAR for table shading
ShadingType.SOLID (causes black cells)
Pill shapes only for tags/chips/badges
Apply large border-radius to input fields
ALL CAPS + letter-spacing for form labels
ALL CAPS for body text or headings
Always show action buttons left-to-right by priority
Mix destructive and primary actions without separation
